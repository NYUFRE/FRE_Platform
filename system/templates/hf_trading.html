HFT

This is a HFT system where users could test order imbalance strategy.
Cleaning Data From DataBase

1) Auto merge the quote data

2) Create the label data, y defined as the average mid-price change with forecast window k = 20, responsible variables as lagged order imbalance.

Linear regression methodology

1) regress average mid-price change with forecast window k = 10 on the six lagged order imbalance

2) Calculate Average coeffcient, Percent positive, Percent positive and signiﬁcant and Percent negative and signiﬁcant

Backtest the strategy

1) Create long position if forecasted average mid-price change is greater than 0

2) Close the position if the loss is larger than threshold or profit is greater than 0

3) We only create the position when there is no any opening position

Reference

1) XU, JIANGMIN. Optimal Strategies of High Frequency Traders.

2) Shen, Darryl. Order Imbalance Based Strategy in High Frequency Trading.
