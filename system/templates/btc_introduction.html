Bitcoin Algorithm Trading System


This module focus on trading Bitcoin with common used algorithms.

Background

Bitcoin is a decentralized currency without the need for a central bank or any middlemen, and was initially presented in 2008. It may be sent to and received from users using the peer-to-peer bitcoin network, where transactions are verified by nodes and stored on a blockchain.
In 2019, Bitcoin was first used after releasing as an open-source software. From then on, Bitcoin continued to be mined by other contributors and was used to buy goods. However, Bitcoin is widely used in black market causing heavy regulation in many countries.

The algorithm trading is an idea to use program to automatically execute orders. In this module, we divided the common used algorithms into three categories: Trend, Momentum and Artificial Intelligence. Each category will include 2 to 3 algorithms.
Algorithms Introduction

Trend

Filter
Filter or Filter Rule is a method that uses price change to determine the trend.
The idea is that once the price changes, it may change in next few days with same trend. For example, a 5% increase in price may let the investor think that the price will increase 10% in next days.
SMA / EMA
Simple or Exponential Moving Average are methods that use moving average to predict the future price.
In this strategy, there are often two moving average lines. One is for the short period and the other is for the long period. The short period line is changing quickly while the long period line is on the opposite.
When the short period line cross above the long period line, this indicates a buy signal and if the short period line cross below the long period line, this indicates a sell signal.
Momentum
MACD
Moving Average Convergence Divergence(MACD) is a method that shows the relationship between two moving average and a signal line.
The MACD line is calculated as the difference of 12 period EMA and 26 period EMA. The signal line is calculated as the 9 period EMA of the MACD line. Then with the same idea of SMA and EMA trading strategy, we determine the buy and sell signal at crossover of MACD and signal line.
RSI
Relative Strength Index(RSI) is a momentum indicator that measures the speed and magnitude of a price changes to evaluate overvalued or undervalued.
When RSI is over 70, it means the price is overbought and investor should sell. When RSI is under 30, it means the price is oversold and investor should buy.
Kalman Filter
Kalman Filter is a method that makes prediction based on previous data. This method contains two stages, The first stage make prediction only based on previous price. The second stage will take the current price into consideration and update the prediction.
As for trading, when the kalman filter prediction line cross above the current price, it indicates a buy signal. When the kalman filter prediction line cross below the current price, it indicates a sell signal.
Artificial Intelligence
LSTM
Long Short-term Memory(LSTM) is a deep learning model that is good for processing time series data. The idea of LSTM is to keep information in each cell, and add or remove state information based on the current input value.
The LSTM model contains three gates, one is the input gate, one is the forget gate, and one is the output gate. The cell will first go through forget gate which decides which information should be removed from the cell state. Then it will go through input gate which decides which information in the current input should be added to the cell state. Finally the output gate will decide which information should be output in the current step.
In trading, the LSTM model can generate prediction of price. We can use the crossover strategy based on prediction and current price.
GRU
Gated Recurrent Unit(GRU) is a deep learning model that is very similar to LSTM which also uses gate to keep information.
However, the GRU does not have cell state, and it uses hidden state to maintain and transfer information. The GRU model contains two gates, which are update gate and reset gate. The reset gate decide how much past information should be removed from the hidden state. The update gate decides how much information should be added to the hidden state and pass to future cell.

Back Testing

In the back testing part, The user has a chance to define parameters in order execution. We will discuss those options in this section:
Initial Capital: This is your initial money that you have to invest.
Execution Type: This parameter contains two options: once and multiple. Once means the investor will only execute one times if notice consecutive signal. For example, if the there are 5 buy signal before a sell signal, we only execute at the first time we notice the signal. Multiple means the investor will execute as long as there is signal. In previous example, we will buy 5 times until selling.
Execution Amount: This means how much each time you will execute.
Price Base: This means which price you want to use for execution. For example, if you use close, then your order will be executed based on the close price.
Position Type: This means which order you want to have. This includes three options: long, short and both. If you choose long, then you will ignore all short signal. If you choose short, the situation will be opposite. If you choose both, then you will execute both long and short signal.
